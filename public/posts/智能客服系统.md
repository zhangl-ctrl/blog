# 系统设计文档

## 1. 系统概述

### 1.1 项目背景

RAG（Retrieval-Augmented Generation，检索增强生成）智能客服系统是一个基于知识库的智能问答系统，通过向量检索技术实现精准的知识匹配，结合大语言模型生成自然流畅的回答。

### 1.2 系统目标

- 提供可视化的知识库管理界面
- 支持多种文档格式的自动导入
- 实现基于 RAG 的智能问答
- 提供多轮对话支持
- 完成后台管理和数据分析

## 3. 功能模块设计

### 3.1 知识库管理模块

#### 功能
- 创建、编辑、删除知识库
- 上传文档（PDF、Word、TXT、Markdown）
- 自动文本提取和分块
- 向量化存储

#### 数据模型
```
KnowledgeBase
├── id: int (主键)
├── name: string
├── description: text
├── created_at: datetime
├── updated_at: datetime
└── is_active: boolean

Document
├── id: int (主键)
├── knowledge_base_id: int (外键)
├── filename: string
├── file_path: string
├── file_type: string
├── file_size: int
├── content: text
├── created_at: datetime
└── is_processed: boolean

TextChunk
├── id: int (主键)
├── document_id: int (外键)
├── knowledge_base_id: int (外键)
├── content: text
├── chunk_index: int
├── vector_id: string
└── created_at: datetime
```

### 3.2 智能问答模块

#### 功能
- 基于 RAG 的问答
- 向量相似度检索
- 引用来源展示
- 多轮对话支持

#### RAG 流程
```
用户输入 → 向量化 → 向量检索 → 上下文构建 → LLM 生成 → 返回回答
```

### 3.3 后台管理模块

#### 功能
- 系统统计面板
- 对话日志查看
- 文档管理

#### 数据模型
```
Conversation
├── id: int (主键)
├── session_id: string
├── user_id: string (可选)
├── created_at: datetime
└── updated_at: datetime

Message
├── id: int (主键)
├── conversation_id: int (外键)
├── role: string (user/assistant)
├── content: text
├── references: text (JSON)
├── created_at: datetime
└── tokens: int (可选)
```

## 4. 核心算法设计

### 4.1 文本分块策略

采用滑动窗口分块策略：
- 块大小：500 字符
- 重叠大小：50 字符
- 尝试在句子边界切分

### 4.2 向量检索策略

使用余弦相似度进行向量检索：
- 检索 top_k = 5 个最相似的文本块
- 过滤条件：knowledge_base_id

### 4.3 回答生成策略

使用 LLM 生成回答：
- 系统提示：基于参考文档回答
- 温度参数：0.7（适度创造性）
- 最大 token：2000

## 5. 接口设计

### 5.1 知识库管理 API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/knowledge/ | 获取知识库列表 |
| POST | /api/knowledge/ | 创建知识库 |
| GET | /api/knowledge/{id} | 获取知识库详情 |
| PUT | /api/knowledge/{id} | 更新知识库 |
| DELETE | /api/knowledge/{id} | 删除知识库 |
| GET | /api/knowledge/{id}/documents | 获取文档列表 |
| DELETE | /api/knowledge/{id}/documents/{doc_id} | 删除文档 |

### 5.2 问答 API

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/chat/ | 发送问题 |
| GET | /api/chat/history/{session_id} | 获取历史记录 |

### 5.3 管理 API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/admin/statistics | 获取统计数据 |
| GET | /api/admin/conversations | 获取对话列表 |
| GET | /api/admin/conversations/{id} | 获取对话详情 |

## 6. 安全设计

### 6.1 文件上传安全
- 文件类型白名单验证
- 文件大小限制（10MB）
- 文件名安全处理（UUID 重命名）

### 6.2 API 安全
- CORS 跨域配置
- 请求超时限制
- 错误信息脱敏

## 7. 性能优化

### 7.1 前端优化
- 组件懒加载
- 虚拟列表（大数据量）
- 请求缓存

### 7.2 后端优化
- 向量数据库索引优化
- 数据库连接池
- 异步处理（文件上传）

### 7.3 LLM 调用优化
- 历史消息截断（保留最近 10 条）
- 超时设置
- 错误重试机制
